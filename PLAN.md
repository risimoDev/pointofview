## 1. Текущее состояние (что работает)

**Прод**: домашний сервер Ubuntu (viziai-server, RTX 3070 8GB) + VPS (ruvds,
AmneziaWG-хаб 10.9.0.1, TLS nginx) + первая боевая точка ПВЗ (10.9.0.11,
4 камеры TP-Link, кассовый ПК Windows 7 как шлюз).

Сделано в этой сессии (всё в main, задеплоено частично — проверить на проде):
- **Live-видео**: go2rtc `<video-stream>` (завендорен, v1.9.14) MSE→HLS,
  один H264-источник на камеру, reconciler в API (60с) чинит go2rtc после
  рестартов. НЕ WebRTC (UDP через VPS — отказались), НЕ MJPEG (тяжёлый, ломался).
- **Статус камер честный**: heartbeat анализатора `camera_alive:{id}` (TTL 15с)
  → GET /cameras вычисляет online/offline. Дашборд поллит 15с.
- **UI полностью на русском**: энумы через `web/lib/labels.ts`.
- **Зоны**: полный CRUD + редактор (пороги dwell и т.д.).
- **События**: resolve, имена камер/зон, снапшоты, клипы.
- **Аналитика** (/analytics): график событий, разбивка по типам/камерам, посетители.
- **Алерты**: Telegram + webhook, тихие часы, мин. severity, тест-кнопка.
- **Re-ID («Сквозная идентификация», фича `reid`)** — НОВОЕ, не проверено на проде:
  - `analyzer/reid/`: embedder (HSV-гистограмма верх/низ тела; интерфейс под
    ONNX OSNet через env `REID_ONNX`) + IdentityManager.
  - Redis: `reid:gallery:{site_id}` (посетители, TTL 12ч), `reid:staff:{tenant}`
    (сотрудники, постоянно), `visitors:{tenant}` (посетители за день ПО ТОЧКЕ).
  - Сотрудники: не считаются посетителями, не триггерят queue_alert /
    zone_violation / crowd. Помечаются на странице /admin/people («Люди»)
    по фото-кропам (MinIO `reid/{tenant}/{gid}.jpg`, через POST /internal/reid/crop).
  - Посетители за день = уникальные личности на точку (конец двойного счёта камер).
  - Миграция `0003_reid_feature.sql`. Пороги в /admin/features.
- **Онбординг ПВЗ**: кит `pvz-onboarding/` — README (пошагово, двойной клик),
  CHECKLIST (там же реальные RTSP-адреса первой точки), `vps/add-site.sh`,
  Windows-скрипты ШАГ-1..5 (PowerShell 2.0-совместимые, UTF-8 BOM обязателен!).
- **Диагностика**: `scripts/diag-camera.sh` (одна камера по всей цепочке),
  `diag-tunnel-camera.sh` (камеры точки с сервера), `diag-stream*.sh`,
  `fix-vps-nginx-ws.sh` (WS-фикс на VPS — ПРИМЕНЯТЬ НА VPS, не на сервере).

## 2. Нерешённые проблемы (проверить в первую очередь)


1. **Re-ID не проверен на проде** — нужен деплой api+web+analyzer, включение
   фичи, прогон: пометить сотрудника, убедиться что посетители дедуплицируются.
   Пороги (0.88/0.90) наверняка потребуют подстройки по месту.
   ОБНОВЛЕНО 2026-07-13 (двойной счёт: «2 курьера ночью = 44 посетителя»):
   - причина: HSV-гистограммы слепнут на ночной ИК-картинке + личность
     создавалась с одного кадра при мерцании трекинга;
   - фиксы в analyzer/reid: пробация (min_samples=3, min_track_age=3с),
     фильтры качества (min_confidence, aspect ratio, min_crop 64px),
     гейт по насыщенности ночью (min_saturation=25, только для гистограмм),
     pending-треки не считаются посетителями (counter.py);
   - «Люди»: отметка сотрудника поглощает его дубли в галерее (cosine ≥
     match_threshold), добавлена кнопка «Удалить» личность;
   - нормальное решение ночи — OSNet ONNX: docs/REID-OSNET.md (экспорт,
     деплой, пороги 0.70/0.75 вместо 0.88/0.90), onnxruntime уже в
     requirements, REID_ONNX + /models уже в prod compose.
2. Деплой: `git pull && docker compose -f infra/docker-compose.prod.yml
   --env-file infra/.env.prod up -d --build api web analyzer` + миграции
   применяет `scripts/update.sh`.

## 3. Принятые решения (не пересматривать без причины)

- Транспорт видео: **MSE+HLS через /go2rtc/ прокси**, задержка 1-3с ок.
- Онбординг владельцев: **инвайт-ссылки** (не свободная регистрация).
- Профили организаций: **все 4** (ПВЗ полный, производство/ритейл/офис — частично dev).
- Шлюз точки: **Windows-ПК точки** (дёшево), но Win7 хрупок → для проблемных
  точек fallback = роутер/мини-ПК.
- Re-ID **без лиц** (по одежде/силуэту) — сознательно: без биометрии и 152-ФЗ
  проблем. Это козырь для гос/корп клиентов, НЕ менять на face recognition.
- Дизайн: без эмодзи, не-lucide иконки (@tabler), весь UI по-русски.
- `.ps1` с кириллицей — только UTF-8 с BOM (PS 5.1/2.0 иначе ломается).

- Рабочий процесс: код на dev-ПК (Windows) → git push → на сервере
  `git pull` + rebuild. У Claude НЕТ доступа к серверу/VPS/точке — диагностика
  через `.sh`-скрипты, вывод присылает пользователь. Отвечать по-русски.
- Реальные данные первой точки (камеры, RTSP, порты): `pvz-onboarding/CHECKLIST.md`.
- graphify: `graphify query "..."` для вопросов по коду; после правок `graphify update .`.
- Проверки перед коммитом: `npx tsc --noEmit` в api/ и web/,
  `python -m py_compile` для analyzer, парсер PS для .ps1 (см. память windows-ps-encoding).
- Память Claude: streaming-architecture, platform-model, pvz-onboarding,
  workflow-prod-testing, ui-localization, windows-ps-encoding, frontend-design,
