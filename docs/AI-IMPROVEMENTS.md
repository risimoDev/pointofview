# План улучшений ИИ-платформы

Файл — ответ на вопрос «как сделать, чтобы ИИ не делал глупых ошибок, и что
добавить дальше». Обновлять по мере выполнения. Приоритет — сверху вниз.

---

## 0. Как система «учится» (важно понимать)

Мы **не дообучаем нейросети** в классическом смысле — это дорого и не нужно.
Точность складывается из четырёх слоёв, каждый настраивается отдельно:

| Слой | Что делает | Где настраивается |
|---|---|---|
| Детекция (YOLO) | находит людей в кадре | conf-порог, frame_skip |
| Трекинг (ByteTrack) | ведёт человека между кадрами | параметры трекера |
| Идентификация (Re-ID) | «этот человек = тот же, что был» | эталоны + пороги, /admin/features |
| Логика (зоны/алерты) | что считать событием | зоны, правила, /admin/settings |

«Дообучить» = дать системе больше **эталонов** (образцы одежды, фото лиц),
правильные **пороги** и правильную **логику**. Ошибка «сотрудника посчитали
посетителем» — это почти всегда: мало эталонов + один порог на все случаи.

## 1. Сделано 2026-07-13

- **Сотрудник = несколько эталонов одежды** (до 8): каждая пометка на «Люди»
  добавляет образец к карточке, а не создаёт дубль. Совпадение — по лучшему
  из образцов.
- **Face-ID сотрудников** (и только сотрудников — посетители без биометрии):
  фото в карточке + автоматический кроп с камеры → сотрудник узнаётся даже
  в новой одежде. Модели YuNet+SFace (CPU, лёгкие), пороги в /admin/features.
- **Анти-флуд алертов**: входы/выходы больше не алертят (только статистика);
  кулдаун по человеку, а не по камере (4 камеры = 1 алерт); некритичное —
  одной сводкой раз в N минут (настройка), critical — мгновенно.
- **Зоны поумнели**: галочка «не срабатывать на сотрудников» (вкл. по умолч.),
  расписание активности зоны и «ночной режим» (алертить всех, режим охраны).
  Состояние зоны привязано к личности, а не к треку — мигание трекера больше
  не рождает повторные события.
- **Кросс-камерность**: события несут global_id человека; подсчёт посетителей,
  дедупликация алертов и зоны понимают, что камеры смотрят на одну точку.

## 2. Точность распознавания — следующие шаги

1. **OSNet вместо цветовых гистограмм** (docs/REID-OSNET.md — всё готово,
   осталось выполнить). Главный прирост качества Re-ID, особенно ночью.
   После включения: пороги 0.70/0.75, сотрудников отметить заново.
2. **Подстройка порогов по факту** (день наблюдений на проде): staff_threshold
   вниз до 0.85–0.87, если сотрудник всё ещё «мигает» посетителем;
   face_threshold 0.32–0.40 по результатам.
3. **Автосбор образцов сотрудника**: когда сотрудник уверенно узнан по лицу,
   система сама добавляет его текущую одежду в эталоны (утром переоделся —
   к обеду система уже знает новый образ без ручной пометки).
4. **Страница «Качество»** в админке: сколько треков осталось без личности,
   сколько раз сотрудник матчился по лицу/одежде, распределение похожестей —
   чтобы пороги крутить по цифрам, а не наощупь.
5. **Разметка ошибок как обучение**: кнопка «это один человек» на двух
   карточках «Люди» → слияние + системе становится известна пара-эталон.
   (Кнопка «добавить к сотруднику» — уже есть, это её развитие.)
6. **Дообучение YOLO на кадрах точки** (когда накопится архив): выгрузка
   сложных кадров (ночь, окклюзии), разметка, fine-tune yolov8n → заметный
   прирост детекции именно на наших ракурсах. Делать после OSNet — сначала
   дешёвые улучшения.
7. **Топология камер**: автоматически выучивать связки «вышел из камеры 1
   справа → через 2 с появился на камере 2 слева» по статистике переходов
   global_id. Даёт: жёсткую сшивку треков между камерами (меньше потерь
   личности) и карту точки без ручной настройки.

## 3. Анти-шум алертов — следующие шаги

- **VLM-верификация перед critical-алертом** (Блок C плана): локальная
  Qwen2.5-VL смотрит кадр и отвечает «действительно ли человек в запретной
  зоне?» — отсев ложных срабатываний от теней/отражений/манекенов.
- **Алерт с описанием**: та же VLM подписывает кадр человеческим языком
  («у стойки трое, один ждёт более 5 минут»).
- **Умный порог очереди**: динамический (по среднему за прошлые недели),
  а не фиксированный dwell_seconds.

## 4. Новый ИИ-функционал (по ценности для владельца)

1. **Ежедневная сводка в Telegram** (вечером): посетители, пик, события.
   Всё для неё уже есть (метрики + воркер алертов). Быстрая победа.
2. **Тепловая карта** передвижений поверх кадра (Redis-сетка 32×18) — вау-фича
   для показов, без новой модели.
3. **Метрики обслуживания**: среднее ожидание в очереди / время у стойки из
   уже записываемых dwell_sec — «язык денег».
4. **Детекция перекрытия/сдвига камеры** (сравнение гистограммы с эталоном
   кадра) → critical-алерт «камеру закрыли».
5. **Локальный ИИ-контур** (Ollama + BullMQ, RTX 3070 ~6.5ГБ свободно):
   VLM-описания (п.3), живая текстовая сводка, ИИ-чат по аналитике
   (вопрос → SQL → ответ).
6. **Поиск по архиву на естественном языке** (CLIP + pgvector): «покажи,
   когда заходил мужчина в красной куртке».
7. **СИЗ/каски для производственного профиля** (после ПВЗ-стабилизации).

## 5. Правила экспериментов

- Один параметр за раз, сутки наблюдения, фиксировать в PLAN.md.
- Не менять пороги одновременно с включением OSNet/face-id.
- Прод — источник истины: любые «улучшения», не подтверждённые падением
  числа дублей/ложных алертов на проде, откатываются.
